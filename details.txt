# Feluda AI - High-Fidelity Technical Specifications (SOC-Grade)

## 0. What is Feluda? (Simple Explanation)
Imagine Feluda as a high-tech security team protecting you while you browse the web. Here is how it works:

1. **The Extension (The Scout):** While you browse, this scout checks every link you see on sites like Google or Facebook. If a link is safe, it shows a **Green Tick**. If it's a dangerous phishing site, it blocks it instantly with a **Quarantine Screen**.
2. **The Backend (The Brain):** When the scout finds a link, it sends it to our AI Brain. The Brain uses artificial intelligence to look for "hacker DNA" (weird characters, fake forms, or hidden scripts) and checks it against global malware databases.
3. **The Dashboard (The Control Center):** This is your main mission hub. You can see a live map of all threats stopped, manually scan suspicious links, or use **File Armor** to scan any file (up to 1GB!) in seconds by checking its "Digital Fingerprint."

## 1. System Vision: The "Neural Node" Concept
Feluda AI is not a passive scanner; it is a **Tactical Cybersecurity Mesh**. It treats every browser as a "Neural Node" that participates in a distributed intelligence grid. By combining local ML-inference with cloud-based forensic telemetry, Feluda achieves a defense-in-depth profile previously reserved for corporate SOCs (Security Operation Centers).

## 2. Advanced Multi-Layer Architecture
The engine (Neural Node v2.5) follows a deterministic tiered analysis flow:

### Layer 1: Structural DNA (URL Analysis)
*   **Entropy Scoring:** Measures the unpredictability of the URL string (Phishing links often have high entropy due to obfuscation).
*   **Homoglyph Filter:** Detects Punycode and visual character spoofing (e.g., Cyrillic 'а' replacing Latin 'a').
*   **TLD Risk Mesh:** Dynamic weighting for high-risk TLDs (.xyz, .tk, .zip, .click).
*   **Identity Age:** Real-time WHOIS correlation to flag "Freshly Minted" domains (0-14 days).

### Layer 2: Machine Learning (Random Forest v2)
*   **Calibrated Probability:** Uses a Platt-scaled Random Forest classifier for high-precision risk scores.
*   **Feature Importance:** 18+ features including digit-to-length ratio, subdomain depth, and keyword density.
*   **Explainable AI (SHAP):** Provides a transparent "Reasoning" field for every block (e.g., "Matched Phishing DNA Pattern X").

### Layer 3: Behavioral DOM Probing
*   **Form Harvesting Detection:** Scans for `<input type="password">` fields posting to external or rogue domains.
*   **Anti-Analysis Radar:** Detects scripts that attempt to block debuggers or hide code from security analysts.
*   **Hidden Infrastructure:** Flags invisible `iframes` or script blocks loading malware payloads.

### Layer 4: Intelligence Grid Integration
*   **Global Reputation:** Real-time sync with Google Safe Browsing and VirusTotal (Aggregated Verdict).
*   **Visual Similarity (Heuristic):** Targeted brand checks for over 50+ financial and social platforms.

## 3. The "Link Shield" Interface
*   **Neural Verified Ticks:** Real-time injection of high-fidelity "Verified" badges in search results (Google, Bing, Facebook).
*   **Mutation-Aware Scanning:** Uses `MutationObserver` to protect dynamic Single-Page Applications (SPAs) without page reloads.
*   **Quarantine Handshake:** A preemptive blocking screen that requires a manual "Override" for high-risk URLs, including a full forensic risk breakdown.

## 5. Operational Flow: How Feluda Protects (Step-by-Step)
Feluda operates as a **closed-loop security ecosystem**. Here is the sequence of events from detection to neutralization:

### Phase A: Edge Detection (The Browser Extension)
1.  **Omni-Monitor:** The extension monitors every URL requested by the browser and scans for `MutationObserver` events on the DOM.
2.  **Reputation Lookup:** Before a page fully loads, the extension sends the URL to the **FastAPI Intelligence Mesh**.
3.  **Visual Injection:** If the site is safe, a **Neural Verified Badge** is injected next to the link (on sites like Google or Facebook).
4.  **Preemptive Interception:** If the backend classifies the site as `Malicious`, the extension halts the navigation and redirects the user to a **Neural Quarantine** screen.

### Phase B: Core Inference (The FastAPI Backend)
1.  **Identity Sweep:** The backend checks the domain against the **Global Blacklist** and performs a WHOIS/ASN verification.
2.  **ML Analysis:** The URL string is parsed through the **Random Forest v2** model to identify structural DNA patterns.
3.  **Behavioral Probe:** The backend "fetches" the target page's HTML to scan for password harvesting forms and external script injections.
4.  **Cloud Sync:** Final verdicts are cross-checked with **VirusTotal** and **Google Safe Browsing** APIs for high-fidelity consensus.

### Phase C: Strategic Visibility (The SOC Dashboard)
1.  **Telemetry Sync:** Every scan event (Safe or Malicious) is logged to **MongoDB Atlas** and streamed live to the **Dashboard**.
2.  **Manual Investigation:** Users can use the **Manual Scanner** or **File Armor** to perform deep-dives into suspicious artifacts.
3.  **System Governance:** The dashboard allows admins to toggle **Failsafe Mode** or update the Global Blacklist in real-time.
